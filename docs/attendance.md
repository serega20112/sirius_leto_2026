# Логика посещаемости

Посещаемость в проекте считается не отдельным cron-процессом и не ручным журналом, а прямо в `TrackAttendanceUseCase`.

## Когда ученик считается пришедшим

Факт прихода фиксируется только после двух условий:

1. система стабильно распознала `student_id`;
2. человек пробыл в кадре не меньше `PRESENCE_CONFIRMATION_SECONDS`.

Это сделано специально. Иначе в базе быстро появляются ложные записи от случайных совпадений на одном-двух кадрах.

## Когда ученик считается опоздавшим

Время начала урока берется из `.env`:

```env
LESSONS_BEGINNING=09:00
```

Дальше use case сравнивает текущее время с этим значением. Если человек отмечен позже, чем:

`lesson_start + ATTENDANCE_LATE_AFTER_SECONDS`

то запись идет с `is_late=True`.

Пример:

- начало урока: `09:00`
- допуск без опоздания: `60` секунд
- человек появился в `09:01:20`
- результат: `late`

## Почему ученик не пишется в базу по десять раз

Тут срабатывают сразу две защиты:

- `marked_students` не дает отметить одного и того же ученика второй раз в тот же день;
- `ATTENDANCE_LOG_COOLDOWN_SECONDS` отсекает повторные записи на коротком промежутке.

На практике это значит, что в обычном режиме по одному ученику в день создается одна основная запись.

## Откуда берутся пропуски

Карточка ученика не опирается на внешнее расписание. Она берет:

- все логи группы;
- строит список дат, когда в группе уже был хотя бы один attendance-log;
- сравнивает этот список с логами конкретного ученика.

Если на такую дату у ученика записи нет, в карточке это считается пропуском.

## Ограничение, про которое лучше знать

Если в конкретный день не распознался вообще никто из группы, то такого дня не появится в истории группы. Соответственно, карточка ученика тоже не покажет этот день как пропуск.

Для школьного режима с настоящим расписанием это решается отдельной сущностью занятия или календарем уроков. В текущем проекте этого слоя нет.

## Что возвращается во frontend

Для главной страницы backend отдает короткий журнал:

- имя ученика;
- timestamp;
- пришел или опоздал;
- уровень вовлеченности.

Для карточки ученика backend считает:

- общее число учебных дней;
- число посещенных дней;
- число приходов вовремя;
- число опозданий;
- число пропусков;
- процент посещаемости;
- список дат опозданий с временем прихода;
- список дат отсутствия.

## Где лежит эта логика

- запись лога: `src/backend/use_case/track_attendance.py`
- агрегирование журнала: `src/backend/use_case/get_report.py`
- сборка статистики по ученику: `src/backend/use_case/get_student_attendance.py`
