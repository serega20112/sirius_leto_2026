# Фронтенд

Фронтенд здесь нарочно простой. Это не SPA и не набор сложных компонентов. Все страницы рендерятся через Jinja2, а поверх уже навешивается немного JavaScript.

## Страницы

### Главная

Шаблон: `src/frontend/templates/index.html`

Что на ней есть:

- боковое меню;
- видеопоток с `/video_feed`;
- журнал посещаемости, который обновляется через `/logs`.

Логику обновления держит `src/frontend/static/js/base.js`. Скрипт каждые две секунды ходит за `/logs`, сортирует записи по времени и показывает последние двадцать.

### Регистрация

Шаблон: `src/frontend/templates/register.html`

Скрипт `src/frontend/static/js/register.js`:

- перехватывает submit;
- проверяет, что заполнены имя, группа и выбрано ровно три фото;
- отправляет `FormData` на `/register`;
- показывает модальное окно с результатом.

### Группы

Шаблон: `src/frontend/templates/groups.html`

Скрипт `src/frontend/static/js/groups.js`:

- раскрывает карточку ученика по кнопке;
- делает запрос на `/students/<id>/attendance`;
- рисует summary, список опозданий и список пропусков.

## Что фронтенд не делает

- не хранит состояние в local storage;
- не редактирует attendance напрямую;
- не рендерит графики через сторонние библиотеки;
- не знает ничего про внутренние пороги AI.

Это нормальное ограничение для текущего проекта. Фронт здесь только показывает результат backend-а.

## Почему это удобно в поддержке

Когда ломается логика посещаемости или AI, почти всегда нужно смотреть backend и консольные логи. Фронтенд в проекте достаточно тонкий, поэтому разбираться в проблемах проще: меньше мест, где может спрятаться ошибка.
