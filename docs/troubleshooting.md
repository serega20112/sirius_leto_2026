# Разбор проблем

Ниже собраны вещи, на которые проект уже реально натыкался.

## Камера не открывается

Симптомы:

- пустой видеопоток;
- ошибки OpenCV в консоли;
- приложение живое, но кадр не обновляется.

Что проверить:

1. Не занята ли камера другим приложением.
2. Правильный ли `CAMERA_SOURCE`.
3. Открывается ли видеофайл напрямую в плеере, если вместо камеры используется путь.

## Лицо не распознается, хотя регистрация прошла

Сначала смотреть на лог при старте:

- `Face gallery refreshed: 0 embeddings` — галерея не собралась;
- `Face gallery refreshed: 3 embeddings` — база лиц есть, проблема уже в live-потоке.

Если галерея пустая:

- проверить, что файлы реально лежат в `src/backend/assets/images`;
- убедиться, что имя файла имеет вид `<student_id>_1.jpg`;
- убедиться, что изображения не повреждены.

Если галерея не пустая, но человек остается `Unknown`:

- ослабить `FACE_DISTANCE_THRESHOLD`;
- уменьшить `FACE_MIN_STABLE_VOTES`;
- проверить, что лицо не слишком мелкое в кадре.

## `facenet-pytorch` не поднимается

Частая история на Windows — битый кэш весов. Признак узнается сразу:

`unexpected EOF ... The file might be corrupted`

Лечится просто:

1. удалить файл из `%USERPROFILE%\.cache\torch\checkpoints`;
2. запустить приложение еще раз;
3. дать модели спокойно докачаться.

Если `PyPI` за прокси не работает, проще ставить пакет напрямую с GitHub:

```powershell
python -m pip install --no-deps "git+https://github.com/timesler/facenet-pytorch.git"
```

## Вовлеченность снова ушла в `unknown`

Смотреть надо на backend, который выбрался при старте:

- `engagement backend: yolo_pose`
- `engagement backend: heuristic_face`
- `engagement backend: mediapipe_legacy`
- `engagement backend: mediapipe_tasks`

Если используется `yolo_pose`, а на части кадров оценка исчезает, это не обязательно поломка. Pose-модель может не собрать keypoints на тесном или неудобном кадре. Сейчас в этом случае estimator откатывается к face-based варианту, поэтому чаще всего вместо `unknown` будут нормальные `low`, `medium` или `high`.

## YOLO работает на CPU, хотя в машине есть видеокарта

Проверять нужно не наличие GPU в системе, а именно то, что видит `torch`.

Простейшая проверка:

```powershell
python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'cpu')"
```

Если здесь `False`, никакие правки в проекте сами по себе GPU не включат.

## `DeepFace` все равно оказался на CPU

Это ожидаемо, если recognizer откатился на запасной backend. В такой конфигурации проект остается рабочим, но распознавание лица не использует CUDA. Причины обычно две:

- `facenet-pytorch` не установлен;
- веса `facenet-pytorch` скачались с ошибкой.

## Опоздание не считается

Проверить нужно три вещи:

1. значение `LESSONS_BEGINNING` в `.env`;
2. значение `ATTENDANCE_LATE_AFTER_SECONDS`;
3. системное время на машине.

Если урок начинается в `09:00`, а запас на вход в класс задан `60` секунд, то запись в `09:00:40` не будет считаться опозданием. Запись в `09:01:10` уже будет.

## В карточке ученика не видно пропусков за день, когда в группе никого не распознали

Это текущее ограничение модели данных. История пропусков строится по датам, которые уже есть в group logs. Если группа в этот день вообще не оставила записей, карточка не узнает, что это был учебный день.

Для исправления нужен отдельный слой расписания.
